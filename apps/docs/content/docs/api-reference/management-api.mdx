The Management API runs on port `:9090` and is used for managing active tunnels.

> [!CAUTION]
> This API requires no authentication and is for internal use only. Do not expose to the public internet!

## Basic Information

- **Base URL**: `http://localhost:9090`
- **Content-Type**: `application/json`

---

## GET /tunnels

List all active tunnels.

### Response

```json
{
  "tunnels": [
    {
      "subdomain": "abc123",
      "user_id": "user_xxx",
      "client_ip": "192.168.1.100",
      "connected_at": "2024-01-15T10:30:00Z"
    }
  ]
}
```

### Field Description

| Field | Type | Description |
|-------|------|-------------|
| `subdomain` | string | Tunnel subdomain |
| `user_id` | string \| null | Authenticated user ID, `null` for anonymous connections |
| `client_ip` | string | Client IP address |
| `connected_at` | string | Connection time (ISO 8601) |

### Example

```bash
curl http://localhost:9090/tunnels
```

---

## DELETE /tunnels/:subdomain

Force disconnect a specific tunnel.

### Path Parameters

| Parameter | Type | Description |
|-----------|------|-------------|
| `subdomain` | string | Subdomain of the tunnel to disconnect |

### Success Response (200)

```json
{
  "success": true,
  "message": "Tunnel 'abc123' disconnected"
}
```

### Error Response (404)

```json
{
  "error": "Tunnel not found: abc123"
}
```

### Example

```bash
curl -X DELETE http://localhost:9090/tunnels/abc123
```

---

## Health Check

The Management API does not yet provide a dedicated health check endpoint. You can verify service status via `GET /tunnels`.

```bash
# Simple health check
curl -s http://localhost:9090/tunnels > /dev/null && echo "OK" || echo "FAIL"
```
